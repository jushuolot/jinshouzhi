# 订单追踪与零成本沟通方案

## 1. 快递式节点追踪

每个订单在 `status_timeline` 记录节点事件：

```json
{
  "key": "in_service",
  "at": "2026-06-12T08:15:00.000Z",
  "detail": "已到场签到，开始服务",
  "actor": "student"
}
```

节点与现有状态机一致：待支付 → 待接单 →（外出审批）→ 待服务 → 服务中 → 待确认 → 已完成。

订单详情页纵向展示：**节点名 + 时间 + 说明**，类似快递物流。

## 2. 注册：真实手机号 + 微信号

| 字段 | 用途 | 是否对外展示 |
|------|------|----------------|
| 手机号 | 登录、平台风控 | 订单对方不可见 |
| 微信号 | 运营审核、异常联系 | 订单对方不可见 |

注册时必填微信号；手机号在登录步骤已绑定。

## 3. 零成本「虚拟联系」——订单密聊（推荐）

**不采购**运营商/云通信的虚拟号、也不对接第三方 IM 服务商。

### 原理（类外卖「联系骑手」，但用 App 内通道）

| 外卖常见做法 | 暖伴替代（零成本） |
|--------------|-------------------|
| 虚拟号转接通话 | **订单内文字 + 语音密聊**（PocketBase/Mock 存消息） |
| 双方见虚拟号 | 双方只见 **代号**：`陪护同学·林` / `家属·联系人` |
| 订单结束销毁号码 | 订单进入 `completed` 后 **自动关闭**密聊入口 |

### API

- `GET /api/nuanban/orders/{id}/messages` — 拉取消息
- `POST /api/nuanban/orders/{id}/messages` — 发送文字或语音（仅服务进行中窗口）
  - 文字：`{ "type": "text", "body": "…" }`
  - 语音：`{ "type": "voice", "audioBase64": "…", "durationSec": 12, "mimeType": "audio/mpeg" }`（最长 60 秒）

订单密聊页提供 **按住说话**，方便老人端；语音以 base64 存入内存通道（零成本，重启 PB 后清空，与文字消息一致）。

### 开放窗口

`pending_accept` → `pending_service` → `in_service` → `pending_confirm`

完成后通道关闭，历史消息运营可按需审计。

### 后续可扩展（仍可不花钱）

- 小程序订阅消息提醒「对方发来新消息」
- 订单内一键复制 **平台客服** 微信（非对端微信号）
- 若未来有预算，再换云虚拟号；**接口层可保持不变**，仅增加「拨打电话」按钮

## 4. 明确不做（当前阶段）

- 真实短信验证码（硬约束跳过）
- 第三方虚拟号 / 隐私号 API（有月租与对接成本）
- 订单双方互显真实手机号 / 微信号
