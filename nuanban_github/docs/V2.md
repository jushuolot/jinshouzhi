# 暖伴勤工 · V2 架构（用户端 / 运营端分离）

> V2 将 **用户端** 与 **运营后台** 拆成两个独立 H5 程序，分别部署到子域名；地图使用 **OpenStreetMap** 免费开源瓦片（无需 API Key）。
>
> **新手请先看：[V2_WALKTHROUGH.md](./V2_WALKTHROUGH.md)**（本地走通 → GitHub → 阿里云子域名，分阶段教程）

## 架构一览

| 程序 | 公网域名 | 本地开发 | 构建变体 | 入口 |
|------|----------|----------|----------|------|
| **用户端** | `https://user.nuanbao.cc` | `http://localhost:5174` | `user` | `/#/pages/common/login` |
| **运营台** | `https://control.nuanbao.cc` | `http://localhost:5175` | `control` | `/#/pages/common/ops-gate` |
| **API** | 两子域同域 `/api` | `localhost:8090`（Vite 代理 `/api`） | — | PocketBase |

根域 `nuanbao.cc` / `www.nuanbao.cc` 可 301 跳转到 `user.nuanbao.cc`（见 `Caddyfile.prod.example`）。

## 一、子域名怎么设置（阿里云上线时用 · 本地可跳过）

> 本地开发不需要 DNS。先 `./scripts/start-v2-local.sh` 按说明用 5174/5175 即可。  
> 完整分阶段教程见 **[V2_WALKTHROUGH.md](./V2_WALKTHROUGH.md)**。

假设主域名已在阿里云备案，服务器公网 IP 为 `101.200.128.82`（以你实际 IP 为准）。

### 步骤 1：登录域名控制台

1. 打开 [阿里云域名控制台](https://dc.console.aliyun.com/)
2. 找到 `nuanbao.cc` → 点击 **解析**

### 步骤 2：添加两条 A 记录

| 记录类型 | 主机记录 | 记录值 | TTL |
|----------|----------|--------|-----|
| A | `user` | `101.200.128.82` | 600 |
| A | `control` | `101.200.128.82` | 600 |

保存后等待 1～10 分钟生效。本地可验证：

```bash
dig +short user.nuanbao.cc
dig +short control.nuanbao.cc
# 均应返回你的服务器 IP
```

### 步骤 3：安全组放行 80 / 443

阿里云 ECS → **安全组** → 入方向确保：

- TCP **80**（HTTP，Caddy 申请证书用）
- TCP **443**（HTTPS）

### 步骤 4：配置服务器环境变量

在服务器 `config/formal.env` 增加（本地可复制 `config/formal.env.example`）：

```env
NUANBAN_DOMAIN=nuanbao.cc
NUANBAN_USER_HOST=user.nuanbao.cc
NUANBAN_CONTROL_HOST=control.nuanbao.cc

# 正式环境：禁用「演示定位（上海）」回退，须浏览器授权真实 GPS
VITE_MAP_REAL=true
```

### 步骤 5：部署 V2

在服务器项目目录执行：

```bash
./scripts/deploy-v2.sh
```

脚本会：

1. 根据 `Caddyfile.prod.example` 生成含 `user.` / `control.` 的 Caddy 配置
2. 分别构建 `h5-user` 与 `h5-control` 两个静态包
3. 重启 Caddy + PocketBase

### 步骤 6：验收

| 地址 | 预期 |
|------|------|
| `https://user.nuanbao.cc/#/pages/common/login` | 用户登录，**无**运营台隐藏入口 |
| `https://control.nuanbao.cc/#/pages/common/ops-gate` | 运营口令门 |
| `https://user.nuanbao.cc/api/health` | `{"code":200,...}` |

浏览器强刷：`Cmd+Shift+R`（Mac）或 `Ctrl+Shift+R`（Windows）。

### 步骤 7（可选）：根域跳转

`Caddyfile.prod.example` 已含 `nuanbao.cc` → `user.nuanbao.cc` 的 301。若仅需子域、不要根域站，可删掉对应 block。

## 二、地图（OpenStreetMap · 免费开源）

H5 端使用 [OpenStreetMap](https://www.openstreetmap.org/) 瓦片，**无需申请 Key、无商用授权费用**：

| 能力 | 实现 |
|------|------|
| 附近老人地图 | `OsmMap.vue` + `tile.openstreetmap.org` |
| 服务区域描点 | `ServiceAreaMapPicker.vue` |
| 老人位置选点 | `ElderLocationMapPicker.vue` |
| 城市 geocoding | Nominatim（`geocode-city.ts`） |

正式环境在 `config/formal.env` 设置 `VITE_MAP_REAL=true` 后重新部署，定位将使用浏览器真实 GPS，不再回退「演示定位（上海）」。

微信小程序等非 H5 端仍使用原生 `<map>`（平台自带地图）。

**注意**：OSM 公共瓦片有访问频率限制；若日活显著上升，可自建 tile 缓存或换商业 CDN，前端组件无需改。

## 三、本地开发

```bash
# 0. 后端 + 测试数据（首次）
./scripts/dev-test.sh

# 1. 查看双端启动说明
./scripts/start-v2-local.sh

# 2. 终端 A · 用户端
./scripts/start-h5-user.sh
# → http://localhost:5174/#/pages/common/login

# 3. 终端 B · 运营台
./scripts/start-h5-control.sh
# → http://localhost:5175/#/pages/common/ops-gate

# 仍可用合一包（兼容旧流程）
./scripts/start-h5.sh
```

构建：

```bash
./scripts/build-h5-variant.sh user
./scripts/build-h5-variant.sh control
```

## 四、与 V1 的差异

| 项 | V1 | V2 |
|----|----|----|
| 运营台 | 嵌在用户 H5，登录页点「暖」进入 | 独立 `control.nuanbao.cc` |
| 用户 H5 | `www.nuanbao.cc` 合一包 | `user.nuanbao.cc` 无运营路由 |
| 地图 | OSM 瓦片 + 可选上海演示定位 | OSM 瓦片 + `VITE_MAP_REAL` 真实 GPS |
| 构建产物 | `dist/build/h5` | `h5-user` + `h5-control` |

GitHub Pages 发布版仍为合一包（`release` 渠道），待 V2 稳定后可拆为 `user` 子路径或独立仓库。
