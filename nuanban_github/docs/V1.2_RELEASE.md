## 相对 v1.1 变更摘要

### 交互与体验（v1.2）

- **全站 Toast 统一**：新增 `utils/toast.ts`，登录页与三端首页提示时长、图标一致
- **登录页**：手机号/验证码实时校验提示；登录按钮仅在信息完整且已勾选协议时高亮
- **三端首页加载态**：学生 / 老人 / 家属 / 运营概览进入与刷新时显示轻量 loading 条
- **触控反馈**：快捷入口、统计卡、待办卡片统一 `nb-tile` / `nb-stat-tap` 按压反馈
- **学生首页**：附近老人空状态 + 跳转发现页；加载失败可点击重试
- **家属首页**：统计卡可点击跳转（绑定 / 待支付 / 外出审批）；空绑定卡片增加「立即绑定」
- **身份选择**：各角色增加功能说明文案；切换身份时显示「正在进入…」
- **主题 token**：`theme.css` 补充 loading 动画、表单 hint、统计卡 tap 样式

### 体检与文档

- `sync-check.sh`：本机 PocketBase 未运行时跳过 API 冒烟（见下方已知限制）
- GitHub Pages 发布渠道已对齐真实 API（非默认 Mock）；仅游客 + `VITE_DEMO_MOCK=true` 走演示数据
- 新增 `V1.2_VERIFICATION_CHECKLIST.md` 逐项验收清单

### 延续 v1.1 能力（未回退）

- 正式鉴权、PocketBase 真实数据路径
- 运营：服务目录、学校合作、派单、老人档案
- 资金持久化与 `isDemoMock` 门禁

## 已知限制（v1.3 候选）

- 微信登录 / 支付仍为演示或 stub
- GitHub Pages HTTPS 访问 HTTP API 可能 mixed-content（备案后 `https://nuanban.cc/api`）
- 本机体检时 PocketBase 未启动则无法跑完整 API 冒烟
- 结算历史订单 backfill 脚本可选，未默认执行
