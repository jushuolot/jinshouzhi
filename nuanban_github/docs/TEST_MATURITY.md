# 暖伴勤工 · 产品成熟度测试清单

> 零成本：公网 GitHub Pages + `demo-mock` 富数据；本地 PocketBase + `seed-demo`。

## 数据集规模（富数据演示）

| 维度 | 数量 | 用途 |
|------|------|------|
| 老人 | 8 | 发现列表滚动、地图多点、双机构 |
| 陪护学生 | 6 | 找陪护列表、预约 |
| 订单 | 20+ | 全状态覆盖、待接单压力 |
| 服务日志 | 8+ | 日志列表、收入关联 |
| SOS | 1 条种子 + 可新增 | 待办横幅 |

## 多角度测试矩阵

### 1. 功能链路

| # | 角色 | 路径 | 通过标准 |
|---|------|------|----------|
| F1 | 学生 | 接单(6单) → 接受 → 签到 → 完成 | 时间轴推进，收入+日志增加 |
| F2 | 学生 | 发现 8 人列表/地图 | 列表可滚动，callout 可点 |
| F3 | 家属 | 订单全部/待支付 → 详情 → 支付 | 状态正确，待接单增加 |
| F4 | 家属 | 外出审批(2单) | 同意/拒绝后状态变化 |
| F5 | 老人 | 绑定码 → 家属输码绑定 | 绑定成功 toast |
| F6 | 老人 | SOS → 家属/学生确认 | 待办消失 |

### 2. 界面展示

| # | 页面 | 检查项 |
|---|------|--------|
| U1 | 学生首页 | 统计数字与待接单角标一致 |
| U2 | 待接单 | `共 N 条` 与卡片数一致 |
| U3 | 收入明细 | 8 条已完成，金额合计合理 |
| U4 | 服务日志 | 与已完成订单对应 |
| U5 | 家属首页 | 绑定 8 人卡片不溢出 |
| U6 | 老人大字号 | 设置开关后首页字号变大 |
| U7 | 桌面预览 | iPhone / 华为 X6 框内无横向滚动 |

### 3. 压力 / 边界

| # | 场景 | 操作 |
|---|------|------|
| P1 | 长列表 | 快速滚动待接单/发现/日志 |
| P2 | 连续接单 | 连续接受 3 单再签到 |
| P3 | 空状态 | 拒绝全部待接单后列表为空提示 |
| P4 | 弱网模拟 | 演示 mock 120ms 延迟仍可操作 |
| P5 | 刷新 | 强刷登录页后数据仍在（内存 mock） |
