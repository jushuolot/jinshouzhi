常见问题
========
• 浏览器提示 ERR_CONNECTION_REFUSED（无法连接 localhost:8501）
  - 说明本机 **没有** 在跑 Streamlit：请先 **双击「启动股票页面.command」**，等终端里出现「You can now view your Streamlit app」再刷新页面；**不要关掉** 该终端窗口。
  - 若终端里有一闪而过的红色报错，说明启动失败，把终端里整段英文复制下来排查。
• 外地朋友在自己电脑浏览器里输入 http://localhost:8501 **永远连不到你家电脑**
  - localhost 只表示「浏览器所在的那一台机器」。要给别人用，请用上文「公网可见」里的 **Streamlit Cloud / VPS / cloudflared / ngrok**，用对方能打开的 **公网链接**，而不是 localhost。

使用方式（仅浏览器）
==================
1. 双击「启动股票页面.command」：会打开终端并启动服务；本机浏览器打开 http://localhost:8501
2. 同一路由器下其他手机/电脑可用：http://你的电脑局域网IP:8501（端口 8501）
3. 首次运行会自动创建 .venv 并安装依赖；使用期间勿关终端窗口。

登录密码（必配）
==================
打开页面前必须先配置 **STOCK_APP_PASSWORD**，否则只能看到「如何配置」的提示：

  方式一：复制 `.streamlit/secrets.toml.example` 为 `.streamlit/secrets.toml`，
         把其中的密码改成你自己的强密码（该文件已加入 .gitignore，勿上传 Git）。

  方式二：启动前在终端执行  
         export STOCK_APP_PASSWORD='你的强密码'

Streamlit Community Cloud：在应用后台 **Secrets** 里增加一行  
`STOCK_APP_PASSWORD = "你的强密码"` 即可。

登录后可在左侧栏点 **「退出登录」**。

公网可见（任选其一，详见下）
==================
【安全提示】已带**访问密码**；公网部署仍建议 HTTPS（Cloud / 隧道自带 HTTPS 更佳）。勿泄露 secrets 与密码。

方式 A — Streamlit Community Cloud（免费托管）
  - 将本项目推到 GitHub，在 https://streamlit.io/cloud 连接仓库；
  - 主文件选 app.py，按向导部署；在 Cloud **Secrets** 中配置 `STOCK_APP_PASSWORD`（与本地 secrets 同名）。

方式 B — 自有 VPS / 云服务器
  - 安全组/防火墙放行 TCP 8501；
  - 服务器上同样创建 venv、pip install -r requirements.txt，执行：streamlit run app.py
  - 浏览器访问 http://公网IP:8501（建议前面再加 Nginx + HTTPS）。
  - 纯 SSH 无桌面时，可设置环境变量：export STREAMLIT_SERVER_HEADLESS=true 以免尝试弹浏览器。

方式 C — 临时内网穿透（本机已能跑 Streamlit 时用）
  - Cloudflare Quick Tunnel（需本机安装 cloudflared）：
      cloudflared tunnel --url http://localhost:8501
  - ngrok（需本机安装 ngrok 并登录）：
      ngrok http 8501
  终端会打印公网 HTTPS 地址，发给他人即可短期访问。
