# 游戏化设计视角（可选产品叙事）

将平台视为 **多人协作 RTS**：LO 是 **关卡任务（Quest）**，仓/车/人是 **单位与建筑**，策略是 **科技树**，异常是 **随机事件卡**，回单与对账是 **通关结算与战利品清点**。

## 三大界面（建议）

1. **世界地图**：风险热力、产能拥塞、SLA 倒计时（运营态势）  
2. **任务队列**：按价值/风险排序的 LO + 一键动作（重算/催办/补证）  
3. **通关结算**：证据规则清单 + 费用解释链（财务闭环）  

## 与工程实现的映射

- Quest = LO + 承诺 + 约束  
- Action = 可调用履约原语（扫码/称重/签收…）  
- Proof = 证据总线（签名/照片/GPS/OCR）  
- Tech Tree = Policy Registry 版本与灰度  

此视角用于 **产品叙事与运营驾驶舱设计**，底层仍以事件账本与投影为准。
