# 产品概述

## 一句话

以 **物流订单（LO）** 统一全渠道、全链条履约；以 **事件账本** 沉淀事实与证据；以 **引擎 + 投影** 实现可重算、可解释、可审计的协同与财务闭环。

## 核心差异化

- **唯一主语**：所有执行与结算围绕 `lo_id`。
- **唯一事实**：`ledger.event` 追加写入；投影可重建。
- **工件可再生**：运单/波次/回单包/对账单由策略与事件生成，而非割裂主数据。
- **决策可审计**：引擎输出写 `DECISION_*` 事件，带策略版本与输入快照引用。

## 多端矩阵

- 平台 Web：LO 队列、LO360、异常、回单、对账、策略、集成与审计  
- 客户 Web / 小程序：下单、可视、回单与对账确认、订阅  
- 承运商 Web：接单、派车、回单、对账  
- 司机 App：任务、在途、签收证据、离线补传  
- 仓 PDA/RF：拣/包/称/装扫码事件  

## 里程碑（建议）

1. **M1**：LO 契约 + 事件账本 + LO Index/LO360 投影 + 权限  
2. **M2**：TaskGraph + 司机/仓事件采集  
3. **M3**：POD/OCR/质检 DSL + 客户确认闭环  
4. **M4**：计费 + 对账解释链 + 结算（可先单边）
