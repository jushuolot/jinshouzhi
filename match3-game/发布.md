# 正式发布：萌植消消乐（GitHub Pages）

仓库里已配置工作流：`.github/workflows/deploy-match3.yml`。按下面顺序做一次即可得到**长期公网链接**（形如 `https://<用户名>.github.io/<仓库名>/`）。

---

## 0. 准备

- 一个 [GitHub](https://github.com/) 账号。
- 本机已安装 Git。工作区路径示例：`/Users/chenli/Downloads/cursor`。

建议新建 **公开（Public）** 仓库：个人账号下，私有仓库的 Pages 在免费档有额外限制；公开仓库最省事。

---

## 1. 在 GitHub 上新建空仓库

1. GitHub 右上角 **New repository**。
2. **Repository name** 自定（例如 `match3-game`），**Public**。
3. **不要**勾选「Add a README」（避免首次 push 要先 pull 的麻烦；若已勾选，见文末「已有 README」）。

记下仓库地址，例如：`https://github.com/<你的用户名>/<仓库名>.git`。

---

## 2. 本地初始化并推送（仅游戏 + 部署配置）

在**工作区根目录**执行（分支名用 `main`；若你惯用 `master`，工作流也已支持）：

```bash
cd /Users/chenli/Downloads/cursor

git init
git branch -M main

git add .github/workflows/deploy-match3.yml match3-game .gitignore
git commit -m "Add match3 game and GitHub Pages deploy"

git remote add origin https://github.com/<你的用户名>/<仓库名>.git
git push -u origin main
```

说明：工作流配置了 `paths` 过滤，只有 `match3-game/**` 或该 workflow 变更才会在 **push** 时自动部署；首次 push 已包含上述路径，会触发一次构建。

---

## 3. 打开 GitHub Pages（必须选 Actions）

1. 打开 GitHub 上该仓库 → **Settings** → 左侧 **Pages**。
2. **Build and deployment** → **Source** 选 **GitHub Actions**（不要选「Deploy from a branch」）。
3. 若第一次使用 Pages，按页面提示完成即可。

---

## 4. 确认部署跑完

1. 打开仓库 **Actions**。
2. 找到 **Deploy match3 (GitHub Pages)**，等待最新一条 **绿色** 成功。
3. 若 GitHub 提示为 **`github-pages` environment** 等待批准：进入该 workflow 运行页，按提示 **Review deployments** → **Approve**（仅首次常见）。

成功后，同一运行页或 **Settings → Pages** 里会显示站点地址，一般为：

`https://<你的用户名>.github.io/<仓库名>/`

用浏览器打开即可游玩（与本地 `index.html` 同源相对路径，无需改代码）。

---

## 5. 以后更新游戏

修改 `match3-game/` 内文件后：

```bash
git add match3-game
git commit -m "Update match3 game"
git push
```

推送成功后 Actions 会自动再部署；刷新公网页即可。

也可在 **Actions** → **Deploy match3 (GitHub Pages)** → **Run workflow** 手动跑一次（无需改代码）。

---

## 可选：自定义域名

在 **Settings → Pages** 的 **Custom domain** 按 GitHub 文档配置 DNS；游戏本身仍为静态文件，无需改构建方式。

---

## 若新建仓库时勾选了 README

首次 push 若被拒绝，先拉再推：

```bash
git pull origin main --rebase
git push -u origin main
```

---

## 其他正式托管（不用本仓库 Actions）

**Netlify / Vercel / Cloudflare Pages**：连接同一 GitHub 仓库，将 **Root directory / 项目根目录** 设为 **`match3-game`**，构建命令留空，发布目录为 `.` 即可。适合需要预览环境、环境变量或全球 CDN 细则时选用。
