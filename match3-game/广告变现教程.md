# 赞助记账一步步教程（古蜀秘档）

玩家免费玩，赞助展示/点击只进入开发者隐藏后台记账。按下面顺序做即可。

---

## 开发者隐藏后台（玩家不可见）

玩家界面**不显示**收益与结算。开发者入口：

1. 在游戏主页**连续点击标题「古蜀秘档」5 次**（2 秒内）
2. 输入口令（默认 `Mz168`，在 `ad-config.js` 的 `admin.passphrase` 修改）
3. 进入后仅可查看本局/累计入账与虚拟余额；禁止提取、扣减或消费

同一次浏览器会话内验证通过后，再次进入无需重复输入口令。

---

打开两个终端窗口。

**终端 A — 游戏页面：**

```bash
cd match3-game
python3 -m http.server 3456
```

浏览器访问：<http://localhost:3456>

你会看到每关开始、结束都有赞助弹窗；玩家界面不显示收益或结算数字。

## 第二步：配置赞助内容（无需外链）

默认已开启 **内置落地页**，不用淘宝客链接也能跑通：

| 文件 | 改什么 |
|------|--------|
| `landing/sponsors.js` | 关头/关尾广告主名称、文案、价格、图标 |
| `ad-config.js` | 开关广告、`minWatchSec`、CPM/CPC 单价 |

本地落地页地址示例：

- 关头：`http://localhost:3456/landing/index.html?slot=level_start`
- 关尾：`http://localhost:3456/landing/index.html?slot=level_end`

上线后自动变成：`https://jushuolot.github.io/game/landing/index.html?slot=...`

### 以后有真实推广链接时

在 `ad-config.js` 中：

```javascript
useBuiltinLanding: false,
advertiser: {
  landingUrl: "https://你的联盟或店铺链接",
  ...
},
```

## 第二步（备选）：只改一个配置文件（有外链时）

用编辑器打开 **`ad-config.js`**，找到 `advertiser`：

```javascript
advertiser: {
  name: "古蜀官方合作伙伴",           // 改成广告主名字
  landingUrl: "https://example.com/shop?...",  // 改成你的真实推广链接
  cpm: 20,   // 每 1000 次展示收入（元）
  cpc: 1.5,  // 每次点击收入（元）
},
```

保存后刷新浏览器即可生效。

### 链接从哪来？

| 来源 | 做法 |
|------|------|
| 自己的店铺/官网 | 直接填商品页或首页 URL |
| 淘宝联盟 / 京东联盟 | 在联盟后台生成推广链接，粘贴到 `landingUrl` |
| 和商家私下合作 | 让对方给你带 `utm_source=match3` 的专属链接 |

建议链接里保留 `utm_source`、`utm_medium` 参数，方便统计流量。

### 关头和关尾要不同链接？

默认两头共用 `advertiser`。若只要关头单独链接，在 `slots.level_start` 里加一行：

```javascript
level_start: {
  label: "关头广告",
  landingUrl: "https://你的关头专属链接",
},
```

## 第三步：理解钱怎么算（演示）

| 动作 | 公式 | 例（cpm=20, cpc=1.5） |
|------|------|------------------------|
| 广告弹窗出现 | `cpm ÷ 1000` | +¥0.020 |
| 点「了解详情」 | `cpc` | +¥1.500 |

数字显示在页面顶部；**累计**存在浏览器本地（清缓存会丢）。

这是给开发者和广告主对账用的**预估**，不是银行自动到账。

## 第四步：把结算记录存到电脑（可选）

若希望每次展示/点击都写入日志文件，用自带的结算服务。

**终端 B — 结算服务：**

```bash
cd match3-game/ad-server
node server.js
```

看到 `http://localhost:3920` 表示已启动。

本地玩游戏时，游戏会**自动**向 `http://localhost:3920/settle` 上报（无需改配置）。

### 怎么确认收到了？

浏览器或终端访问：

- 统计：<http://localhost:3920/stats>
- 最近记录：<http://localhost:3920/log>

原始日志文件：`ad-server/settlements.jsonl`（一行一条 JSON）。

### 上线到 GitHub Pages 后

公网页面不能访问你电脑的 `localhost`。需要：

1. 把 `ad-server` 部署到云服务器（或 Serverless）
2. 在 `ad-config.js` 里写公网地址：

```javascript
settlementWebhook: "https://你的域名/ad/settle",
```

## 第六步：和广告主结算（商业）

1. 约定计价：按 CPM（展示）还是 CPC（点击），或混合

## 常见问题

**Q：能关掉广告吗？**  
`ad-config.js` 里设 `enabled: false`。

**Q：强制看多久？**  
改 `minWatchSec`（秒），倒计时结束才能点「继续游戏」。

**Q：真接 Google AdSense 呢？**  
需要把广告平台的 iframe/脚本嵌进 `index.html` 的 `#ad-frame` 区域，并遵守平台政策；和当前「自建赞助链接」是另一条路。

**Q：玩家跳过广告？**  
不能跳过倒计时；「继续游戏」在倒计时结束前是灰的。

## 文件说明

| 文件 | 作用 |
|------|------|
| `ad-config.js` | **你只改这个**：链接、单价、开关 |
| `game.js` | 广告弹窗、结算逻辑（一般不用动） |
| `ad-server/server.js` | 本地接收结算 POST 的演示服务 |
| `index.html` | 收益栏 + 广告弹窗界面 |
